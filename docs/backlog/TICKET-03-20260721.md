# TICKET-03: Redis healthcheck trả về "N/A"

**Phát hiện:** E2E test ngày 2026-07-21 trên `https://his.atds.com.vn/`

## Severity & Priority
- **Severity:** Low
- **Priority:** P3

## Mô tả
Endpoint `GET /api/v1/health` hiện trả:
```json
{ "database": "OK", "redis": "N/A", "storage": "OK" }
```

Trạng thái `N/A` không phân biệt được **Redis chưa được cấu hình**, **Redis down**, hay **healthcheck chưa implement**. Điều này gây khó khăn cho monitor (Sentry/Grafana) khi cảnh báo sự cố hạ tầng cache/queue.

## User Story
**Given** tôi là DevOps giám sát hệ thống,
**When** tôi gọi `/api/v1/health`,
**Then** trường `redis` phải trả **`OK`** (kết nối thành công) hoặc **`Down`** (không kết nối được) kèm chi tiết lỗi.

## Acceptance Criteria
- [ ] Verify Redis service đang chạy trong `docker-compose.yml` và backend có config `ConnectionStrings:Redis`.
- [ ] Implement healthcheck dùng `AspNetCore.HealthChecks.Redis` (hoặc custom ping `PING → PONG`).
- [ ] Response format:
  ```json
  { "redis": { "status": "OK", "latencyMs": 3 } }
  ```
  hoặc
  ```json
  { "redis": { "status": "Down", "error": "Connection refused" } }
  ```
- [ ] Timeout healthcheck Redis: **2 giây** (tránh block toàn bộ endpoint).
- [ ] Cập nhật Grafana/Sentry alert khi `redis.status = Down` liên tục 3 lần trong 5 phút.
- [ ] Document trong `docs/api/health.md`.

## Assignee
- Backend (implement healthcheck) + DevOps (verify infra + alert)

## Effort
**S** (0.5 ngày)
