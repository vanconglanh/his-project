# TICKET-02: Thiếu endpoint GET /api/v1/auth/me

**Phát hiện:** E2E test ngày 2026-07-21 trên `https://his.atds.com.vn/`

## Severity & Priority
- **Severity:** Low
- **Priority:** P3 (nice-to-have, đưa vào sprint kế tiếp)

## Mô tả
Frontend hiện lấy thông tin user từ **response của `POST /api/v1/auth/login`** và lưu vào state/localStorage. Khi user **F5/refresh trang** hoặc mở tab mới với token còn hiệu lực, FE không có cách re-fetch thông tin user → phải parse JWT thủ công (không an toàn) hoặc bắt user login lại.

Endpoint `GET /api/v1/auth/me` hiện trả về **404 Not Found**.

## User Story
**Given** tôi là user đã đăng nhập với access token hợp lệ,
**When** FE gọi `GET /api/v1/auth/me` kèm `Authorization: Bearer <token>`,
**Then** API trả về thông tin user hiện tại để hydrate lại UI sau refresh.

## Acceptance Criteria
- [ ] Tạo endpoint `GET /api/v1/auth/me` (yêu cầu JWT hợp lệ).
- [ ] Response schema:
  ```json
  { "data": { "userId": 1, "email": "...", "role": "BacSi",
    "tenantId": 1, "clinicId": 1, "permissions": ["patient.read", ...],
    "fullName": "Nguyễn Văn A" } }
  ```
- [ ] Trả `401 UNAUTHORIZED` nếu token thiếu/hết hạn (theo error envelope chuẩn).
- [ ] Cập nhật Swagger/OpenAPI có summary tiếng Việt.
- [ ] Thêm unit test + integration test (happy path + 401).
- [ ] FE tích hợp `useCurrentUser()` hook dùng TanStack Query, cache 5 phút.
