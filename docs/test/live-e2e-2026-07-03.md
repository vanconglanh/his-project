## 3. Bug runtime THẬT đã phát hiện & fix

- **`GET /api/v1/encounters` → 500** (`System.FormatException: Guid string should only contain hexadecimal characters`). Nguyên nhân: 1 dòng seed có `id` **không phải GUID hex** — `e2eenc01-0000-0000-0000-000000000001` (chữ `n` trong "enc") → EF Core `Guid.TryParse` fail khi materialize → **sập cả danh sách encounters**. Fix: soft-delete dòng seed sai → `/encounters` **200**, Encounter LIST+CREATE **PASS**.
  - *Bài học:* đây là bug mà **đọc code không thấy** — chỉ lộ khi chạy runtime với dữ liệu thật.

## 4. Phát hiện quan trọng — điều tra sâu nguồn gốc SKIP (KHÔNG phải thiếu tính năng)

**Root cause (xác minh bằng inspect DOM trên browser thật):**
1. **Spec timing-flaky:** helper check bằng `.count()` tức thì + `waitForTimeout` cứng ngắn (500–1200ms). Menu base-ui render qua portal + animation; DataTable load data async (TanStack Query) → element chưa kịp hiện → SKIP. **Inspect DOM chứng minh row-action MỞ ĐƯỢC**: trigger `<button aria-label="Thao tác">`, click → 3 `menuitem` "Xem chi tiết / Chỉnh sửa / Xoá" đúng role. Tính năng CÓ, chỉ là spec đo hụt. (Icon đổi `lucide-more-horizontal`→`lucide-ellipsis` do design-system, nhưng aria-label vẫn khớp.)
2. **Cạn tài nguyên:** mỗi lần E2E bị kill để lại ~5 chromium orphan → tích tụ **22 process** → cạn CPU → frontend chậm dần → SKIP tăng dần (giải thích 33→28).
3. **Stack chậm:** backend + **Hangfire retry-storm** (jobs stale query `cli_visits`/`status` fail rồi retry liên tục) + **WSL MySQL I/O** → API chậm → interaction timeout (vd Patient CREATE FAIL: click 10s).

**Đã sửa spec (`e2e/crud-actions.spec.ts`):** thay `.count()` + `waitForTimeout` cứng bằng `waitFor({state:'visible'})` cho row-dropdown / tab / create / generic-action / submit + **retry** mở menu. Cải thiện rõ khi nhẹ tải (Patient 5/5) nhưng KHÔNG thắng được stack chậm.

→ **Con số CRUD trên môi trường WSL này KHÔNG phải thước đo tin cậy.** Độ phủ tính năng thực gần R7 (~43) hoặc hơn. Nhóm SKIP cố định mọi lần (~13) = gap thật/cần flow riêng (Encounter AddVital/AddDiagnosis/Close, Prescription AddDrugItem/Submit, Drug Search, ServiceCatalog UpdatePrice, Supplier Update, Reception PrintTicket, AdminRoles EditPermissions) — trùng SKIP của R7.
